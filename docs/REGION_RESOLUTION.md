# Region Resolution

정책 지역은 `NATIONWIDE`와 `UNKNOWN`을 분리한다.

`NATIONWIDE`는 전국 청년, 전국 거주자, 지역 제한 없음, 거주지 제한 없음처럼 정책 대상 문맥에서 전국 근거가 명시된 경우에만 사용한다.

`UNKNOWN`은 지역 정보를 찾지 못했거나 판단 근거가 부족한 상태다. 지역 근거가 없다는 이유로 전국 정책으로 저장하지 않는다.

## 판별 우선순위

1. 정책명
2. 지원 대상·참여 대상
3. 추가 신청 자격
4. 지원 내용·정책 설명
5. `zipCd`
6. 주관 기관
7. 운영 기관
8. 등록 기관
9. 전국 명시 여부
10. 판단 불가

정책명, 지원 대상, 신청 자격, `zipCd`는 강한 근거다. 기관명은 보조 근거이며 정책명이나 지원 대상보다 우선하지 않는다.

## 지역명 추출

긴 지역명부터 매칭한다. `인천광역시 남동구`처럼 구 단위가 명확하면 단독 `인천`보다 구 단위를 우선한다.

약칭은 정규화한다. 예: 서울은 서울특별시, 경기/경기도는 경기도, 경북은 경상북도, 경남은 경상남도.

## 기관명 처리

지방 기관명은 보조 지역 근거로 사용한다. 예: 수원시청은 경기도 수원시, 경기도일자리재단은 경기도.

중앙기관은 지역 근거로 사용하지 않는다. 예: 고용노동부, 보건복지부, 중소벤처기업부, 국토교통부, 교육부, 금융위원회, 한국장학재단, 서민금융진흥원, 한국고용정보원.

## zipCd 처리

`zipCd`는 쉼표로 분리해 `region_code.region_code`와 매칭한다. 수원시 구 코드 여러 개가 들어오면 여러 지역으로 저장하며 scope는 `MULTIPLE`이다. 파싱할 수 없는 코드는 `UNKNOWN`으로 둔다.

## Hard Filter

사용자가 경기도 수원시를 입력하면 다음은 통과한다.

- 전국
- 경기도 전체
- 수원시
- 수원시 구 단위
- 여러 지역 중 수원 포함

다음은 제외한다.

- 서울특별시
- 성남시 전용
- 용인시 전용
- 인천 남동구/부평구
- 충청남도 서산시
- 경상남도 창원시

`UNKNOWN`은 기본 검색 결과에서 제외한다. 필요하면 `RAG_INCLUDE_UNKNOWN_REGION=true`로 포함할 수 있다.

## 점수

지역 입력이 있으면 지역 점수를 크게 반영한다.

- EXACT_CITY, EXACT_DISTRICT: 100
- MULTIPLE_REGION_MATCH: 95
- PROVINCE_MATCH: 80
- NATIONWIDE: 45
- UNKNOWN: 0
- NOT_MATCHED: Hard Filter

정렬은 시/군/구 정확 일치, 광역 지역, 전국, UNKNOWN 순서다.

## 기존 데이터 재계산

관리자 화면에서 `전체 정책 지역 다시 계산`을 실행한다.

API:

```text
POST /api/admin/jobs/policy-region-rebuild
```

흐름:

1. 활성 정책 전체 ID 조회
2. 새 지역 resolver 실행
3. 기존 `policy_region`과 비교
4. 잘못된 전국 관계 제거
5. 실제 지역 관계 추가
6. 변경 정책을 임베딩 `PENDING`으로 등록
7. 관리자 화면에서 전체 PENDING 임베딩 처리

MySQL/Qdrant 볼륨은 삭제하지 않는다.
# SGIS 카탈로그 기반 지역 해석

지역 정규화의 기준 데이터는 `region_code` 테이블이다. 전국 시·군·자치구는 SGIS 공식 단계별 주소 API로 동기화하고, 검색 시에는 외부 API를 호출하지 않는다.

내부 지역 식별자는 외부 코드와 분리한다.

- 전국: `KR`
- 시·도: `P:{정규화된 시도명}`
- 시·군·자치구: `M:{정규화된 시도명}:{정규화된 시군자치구명}`

SGIS `cd`와 온통청년 `zipCd`는 `region_external_code`에 코드 체계별로 저장한다. 숫자가 같다는 이유만으로 서로 같은 지역 코드로 보지 않는다.

사용자 입력은 `UserRegionTextResolver`가 DB 카탈로그에서 해석한다. 사용자 검색에서 쓰는 논리 단위는 `NATIONWIDE`, `SIDO`, `SIGUNGU`, `UNKNOWN`이다.

- `칠곡` / `칠곡군` / `경북 칠곡` → 경상북도 칠곡군
- `횡성` → 강원특별자치도 횡성군
- `예산` → 충청남도 예산군
- `해남` → 전라남도 해남군
- `합천` → 경상남도 합천군
- `광주` → AMBIGUOUS
- `광주광역시` → 광주광역시
- `경기도 광주` → 경기도 광주시

사용자 검색어 해석은 로컬 DB 카탈로그만 사용한다. SGIS API는 검색 요청마다 호출하지 않는다.

DB 저장 레벨은 기존 호환을 위해 `PROVINCE`, `CITY`를 유지하지만, 사용자 검색 화면과 Resolver 결과에서는 `PROVINCE`를 `SIDO`, `CITY`를 `SIGUNGU`로 표현한다. `SIGUNGU`는 도·특별자치도 아래 시·군과 특별시·광역시 아래 자치구·군을 포함한다. 일반시 하위 일반구와 읍면동은 독립 검색 지역으로 저장하지 않고 상위 시·군으로 통합한다.

- 수원시 영통구 → 경기도 수원시
- 청주시 흥덕구 → 충청북도 청주시
- 칠곡군 왜관읍 → 경상북도 칠곡군
- 서울특별시 강남구 → 서울특별시 강남구 유지
- 인천광역시 부평구 → 인천광역시 부평구 유지

## 사용자 검색 지역 우선순위

공식 전체 경로가 가장 우선한다. 그다음 공식 시·도명, 공식 시·군·자치구명, 시·도+시·군·자치구 별칭, 생성 별칭 순서로 후보를 평가한다.

시·도 전체 별칭만 입력된 경우 산하 자치구나 시·군으로 자동 변경하지 않는다. 하위 지역은 하위 공식명이나 부모 시·도와 결합된 별칭이 실제 입력에 있을 때만 선택한다.

동일 우선순위의 후보가 여러 지역에 걸리면 `AMBIGUOUS`로 남기고 지역 Hard Filter를 적용하지 않는다.
